# Security Policy

## Supported Versions

The following versions of Digital-FTE are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Digital-FTE seriously. If you believe you have found a security vulnerability, please report it to us by following these steps:

1. **Do not open a public issue.**
2. Send an email to [muhammadabdullah51700@gmail.com](mailto:muhammadabdullah51700@gmail.com) with the subject "SECURITY VULNERABILITY: [Brief Description]".
3. Include as much detail as possible, including steps to reproduce the issue.

We will acknowledge your report within 48 hours and provide a timeline for a fix. We ask that you follow [responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure) practices.

## Security Best Practices for Users

Digital-FTE is a powerful tool with access to your accounts. To keep your installation secure:

- **Never** commit your `.env` file or any other files containing credentials.
- Use a dedicated, non-primary email account for testing if possible.
- Regularly review the permissions granted to your OAuth applications.
- Keep your local machine and cloud server updated with the latest security patches.
- Use the **Human-in-the-Loop** feature to review all drafts before they are sent.
