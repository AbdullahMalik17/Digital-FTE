## How to use this list

These are curated **good first issues** for new contributors.  
When you create GitHub issues, copy the titles and descriptions below and add the label **`good first issue`** (and `documentation` / `enhancement` where relevant).

---

### 1. Add example Obsidian Vault template

**Goal:** Provide a minimal example Vault structure that matches the folders described in the README (`Needs_Action/`, `Pending_Approval/`, etc.).  
**Why:** New users can get started faster without designing their own folder layout.

Suggested steps:
- Create an `examples/obsidian-vault/` directory mirroring the documented structure.
- Include a short `README.md` inside explaining how the Vault is used by Digital-FTE.

Labels: `good first issue`, `documentation`.

---

### 2. Add screenshots and diagrams to README

**Goal:** Enrich the main README with 2–3 screenshots (dashboard, Vault view, logs) and the existing architecture diagram.  
**Why:** Visuals improve understanding and help judges/users see value quickly.

Suggested steps:
- Capture or generate screenshots of a typical flow.
- Add a “Screenshots” or “Product Tour” section in `README.md`.

Labels: `good first issue`, `design`, `documentation`.

---

### 3. Create CONTRIBUTING guide

**Goal:** Create `CONTRIBUTING.md` with setup, branch naming, testing, and PR guidelines.  
**Why:** Makes it easier for new contributors to follow the project’s workflow.

Suggested steps:
- Document how to run tests (`pytest tests/`).
- Explain coding style expectations and commit message conventions.

Labels: `good first issue`, `documentation`.

---

### 4. Add sample `.env.example` comments

**Goal:** Improve `.env.example` with short comments for each key (what it is and where to get it).  
**Why:** Reduces friction for first-time setup.

Suggested steps:
- For each environment variable, add a short inline comment explaining its purpose.

Labels: `good first issue`, `enhancement`.

---

### 5. Docs: link Mintlify / GitBook site from README

**Goal:** Once the interactive docs are deployed, add a prominent “Docs” link at the top of the README.  
**Why:** Encourages people to use the new documentation site.

Suggested steps:
- Add a badge or button-style link near the top of `README.md`.
- Ensure the link stays in sync with the deployed docs URL.

Labels: `good first issue`, `documentation`, `website`.
