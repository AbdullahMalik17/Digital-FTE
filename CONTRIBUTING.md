# Contributing to Digital-FTE

First of all, thank you for your interest in contributing! It's people like you that make Digital-FTE such a great tool for the community.

## How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** issue template in GitHub (`.github/ISSUE_TEMPLATE/bug_report.yml`).
- Provide as much detail as possible: steps to reproduce, environment, logs, etc.

### Suggesting Enhancements
- Use the **Feature Request** issue template in GitHub (`.github/ISSUE_TEMPLATE/feature_request.yml`).
- Clearly describe the benefit and use case.

### Good First Issues (for new contributors)
- Look for issues labeled **`good first issue`**.
- If you're a maintainer, create issues using the **Good First Issue** template (`.github/ISSUE_TEMPLATE/good_first_issue.yml`).

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style.
5. Provide a clear and concise description of your changes in the PR.

## Style Guide

### Python
- Follow PEP 8.
- Use type hints for all function signatures.
- Document classes and functions with docstrings.

### TypeScript / Next.js
- Use functional components.
- Use Tailwind CSS for styling.
- Keep components small and focused.

### Git Commit Messages
- Use the imperative mood ("Add feature" not "Added feature").
- Keep the first line under 50 characters.
- Use the body to explain what and why vs. how.

## Code of Conduct
We expect all contributors to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?
If you have questions about the project, feel free to open an issue or contact the author.
